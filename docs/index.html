Brief Overview of Tempomeme

Tempomeme is a Tempo Mainnet meme launch platform centered on USDC-settled curve trading, internal sale completion, and a defined platform fee policy around the official token $TME.

Network
Tempo Mainnet
Chain ID
4217
Official Token
$TME

1. Chain

Tempomeme is deployed on Tempo Mainnet. The current frontend, wallet connection flow, token creation flow, and trade flow are all designed around this chain only, so users interact with a single network and a single settlement asset.

Network Parameters

Network: Tempo Mainnet
Chain ID: 4217
RPC: https://rpc.tempo.xyz

Settlement Asset

All current platform trading is settled in USDC on Tempo.
USDC: 0x20c000000000000000000000b9537d11c60e8b50

  • Users need an EVM wallet connected to Tempo Mainnet.
  • Token creation, buying, and selling all run on Tempo.
  • The current launch flow uses USDC as the settlement token for the internal curve.

2. Price Curve and External Migration

The current primary launch contract uses a quadratic pricing curve across the internal sale cap. Price moves upward as sold supply increases, and the project only moves to the external liquidity stage after the internal curve is completely filled.

Primary Launch Contract

0x25d8978d45e8987b55e3E16132eb4a65Bf4Dc6C4

Internal Sale Cap

800,000,000 tokens
This represents 80% of the total supply.

Curve Formula
price(sold) = 0.000003 + 0.000062 * (sold / 800,000,000)^2
Detailed Calculation
Let: saleCap = 800,000,000 r = sold / saleCap Then: price(sold) = 0.000003 + 0.000062 * r^2 Buy side: buyFee = usdcIn * 1% netUsdc = usdcIn - buyFee tokensOut = netUsdc / price(sold) Sell side: grossUsdcBack = tokensIn * price(sold) sellFee = grossUsdcBack * 1% usdcOut = grossUsdcBack - sellFee Migration condition: migrate when sold >= 800,000,000
  • The curve starts near 0.00000300 USDC.
  • Because the pricing model is quadratic, the price accelerates more aggressively later in the sale than at the beginning.
  • The simplified ratio form is r = sold / 800,000,000, so all price movement is driven by sold supply progress on the internal curve.
  • For buys, the 1% fee is removed first, then the remaining USDC is converted at the current curve price.
  • For sells, the token amount is first converted back into gross USDC at the current curve price, then the 1% fee is deducted.
  • External migration is triggered only after 800,000,000 tokens are sold on the internal curve.
  • On a no-sell path, completely filling the curve is about 19,124.58 USDC gross user spend.
  • After the current 1% trading fee, the contract retains about 18,933.33 USDC net.

3. Official Token

The official ecosystem token is $TME. In the current platform description, trading fees are treated as both ecosystem support and operating revenue, with the stated split designed to support the token and the platform at the same time.

Buyback Allocation

50% of trading fees are allocated to buy back $TME.

Operations Allocation

50% of trading fees are reserved for maintenance and project development.

  • The buyback portion is meant to support the official token over time.
  • The development portion is meant to support maintenance, upgrades, and ecosystem growth.
  • This section describes the platform fee policy around $TME.
