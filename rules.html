Rules

This page explains how token creation, pricing, trading, rankings, and live data work across the site.

1. Token Creation
Any connected wallet can create a new launch. Creating a token deploys a new on-chain meme token and registers it in the launch contract.
  • The create form supports name, symbol, description, image, X link, and website.
  • Project metadata is stored in the launch event and used by the site to render token cards and detail pages.
  • New tokens appear in the Home and Tokens pages after the launch is indexed from chain data.
2. Trading and Settlement
All trades on Tempo Pump are settled in USDC. Buy and sell actions interact directly with the launch contract.
  • Buys use USDC and deliver token output based on the current curve price.
  • Sells return USDC based on the current curve price and available contract liquidity.
  • The protocol fee is 1% on both buy and sell transactions.
3. Curve and Graduation
Pricing follows the launch contract curve. Each token progresses along the curve as net USDC is added through buys.
  • Curve progress is displayed as a percentage of the graduation target.
  • The graduation target is 12,000 USDC.
  • When a token reaches the target, the launch is marked as graduated on-chain.
4. Rankings and Token Lists
The site provides token lists on the Home page and the dedicated Tokens page.
  • Lists can be sorted by latest creation order or by market cap.
  • Market cap is derived from live price and token total supply.
  • Cards show live curve progress instead of raised USDC totals.
5. Charts, Activity, and Holders
Token detail pages use on-chain events to build the market view for each token.
  • The price chart is generated from real TokenBought and TokenSold events.
  • Activity tables show only real on-chain trades.
  • Holder tables are derived from token Transfer events and current balances.
6. Live Data Policy
Tempo Pump does not fabricate market activity.
  • The top ticker shows only real trades from chain events.
  • If no recent trades exist, the site shows an empty state instead of fake messages.
  • Wallet balances, prices, and token state are read from live contract data.
7. User Responsibility
Users are responsible for reviewing token information and transaction details before confirming any action.
  • Always check wallet prompts, slippage expectations, and token amounts before signing.
  • On-chain transactions are final once confirmed.
  • Token creation and trading involve market risk, liquidity risk, and smart contract risk.
